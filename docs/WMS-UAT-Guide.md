## 1. 系统模块概览

| 模块 | 入口 | SAP 对应 | 核心操作 | 适用场景 |
|------|------|---------|---------|---------|
| **PO 采购收货** | po.html | 采购订单 OPOR | 扫码 → 录数量 → 提交 | 仓库收到供应商送货 |
| **WO 生产收货** | wo.html | 生产订单 OWOR | 扫码 → 录数量 → 提交 | 车间完工入库 |
| **SO 销售拣货** | so.html | 销售订单 ORDR | 扫码 → 录数量 → 提交 | 仓库按单拣货出库 |
| **PI 生产领料** | pi.html | 生产订单 BOM | 扫码 → 录数量 → 提交 | 按工单 BOM 领料出库 |
| **TR 库存调拨** | tr.html | 转储申请 OWTQ | 扫码 → 录数量 → 提交 | 不同仓库间调拨 |
| **LM 库位移动** | lm.html | 无 (WMS 内部) | 扫码 → 选库位 → 提交 | 同仓库内换库位 |
| **IC 盘点** | ic.html | 无 (WMS 内部) | 扫码 → 录盘点数 → 提交 | 实物盘点 (数量可正可负) |
| **库存查询** | stock.html | — | 查询 | 查看实时库存 |
| **数据导出** | export.html | — | 选择导出 | 导出 CSV 交 SAP 过账 |
| **OMS 订单管理** | oms.html | SO/PO/WO/TR/DD | 订单查询、DD 拆单、批量打印 | 统一订单中心 + DD 调度 |

## 2. 全流程测试用例

### 测试流程总览

```
登录 → 选模块 → 输入/扫描单号 → 加载单据 → 逐行扫物料 → 录入数量 → 提交
→ (所有行完成后) → 导出 CSV → 交 SAP 过账
```

---

### 用例 A: PO 采购收货 (建议优先测试)

**前置**: SAP 中存在未完成的采购订单

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 登录 WMS | 进入首页仪表板 |
| 2 | 点击 "PO 采购收货" | 进入 PO 页面，扫码框自动获取焦点 |
| 3 | 输入 PO 编号并回车 | 从 SAP 加载单据行项目列表 |
| 4 | 扫描/输入物料编号 | 自动匹配对应行并高亮 |
| 5 | 输入收货数量 | 数量框显示输入值 |
| 6 | 选择库位 | 库位下拉显示可选项 |
| 7 | 点击 "提交" | 绿色成功提示，行项目已收数量更新 |
| 8 | 重复步骤 4-7 直到所有行完成 | 所有行显示绿色完成状态 |
| 9 | 进入导出页面 → 筛选 PO 类型 → 导出 CSV | 下载含时间戳的 CSV 文件 |

**验证要点**:
- [ ] 扫码后焦点自动到数量输入框
- [ ] 错误提示后焦点自动回到扫码框
- [ ] 多次扫同一物料时数量累加
- [ ] CSV 中 UOM 列有值 (如 PCS, KG)

### 用例 K: 超量场景 (边界测试)

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 打开已有行的单据，某行已完成 | — |
| 2 | 再次扫描该行的物料 | 弹出确认框 "该行已完成...是否继续录入？" |
| 3 | 点 "取消" | 焦点回到扫码框 |
| 4 | 点 "确定" | 打开数量输入卡片 |
| 5 | 提交超量数量 (需填写备注) | 事务成功，**单据状态保持 in_progress** |

**验证要点**:
- [ ] 超量后单据不会自动变成 "已完成"
- [ ] 取消后焦点回扫码框

## 3. 已知问题与需求分类

### 什么是 BUG？什么是需求？

| 分类 | 定义 | 举例 |
|------|------|------|
| **BUG** | 功能不按预期工作，属于软件缺陷 | "点提交后无响应"、"导出数据为空" |
| **需求** | 功能正常但希望增强或新增 | "希望支持批量扫码"、"想要操作撤销" |
| **体验** | 界面或交互可以更友好 | "按钮太小不好点"、"颜色看不清" |
| **已知限制** | 当前版本的设计约束 | "超量行需手动标记完成" |

### 典型需求方向参考

| 方向 | 说明 | 优先级建议 |
|------|------|---------|
| 数量精度控制 | 按物料 UOM 自动控制小数位 | 中 |
| 单据手动完成 | 超量行的单据添加手动完成按钮 | 高 |
| SAP 过账反馈 | CSV 导出后追踪 SAP 过账是否成功 | 中 |
| 操作撤销 | 误操作后撤销上一次扫码提交 | 中 |
| 批次号管理 | 全面启用批次号/生产日期录入 | 低 |
| 离线支持 | 网络中断时缓存操作，恢复后同步 | 低 |
| 多语言完善 | 四语覆盖所有提示信息 | 低 |

## 4. 如何提交问题反馈

### 反馈模板

```
【类型】BUG / 需求 / 体验 / 疑问
【模块】PO / WO / SO / TR / LM / IC / PI / OMS / 导出 / 库存 / 登录
【操作步骤】
  1. 进入 XX 页面
  2. 输入 XX 单号
  3. 扫描物料 XX
  4. ...
【实际结果】描述看到的现象
【期望结果】描述你认为应该怎样
【截图/录屏】(如有)
【发生时间】
【测试账号】
```

### 反馈示例

```
【类型】BUG
【模块】PO 采购收货
【操作步骤】
  1. 进入 PO 页面
  2. 输入单号
  3. 扫描某物料 (该物料出现在多个行)
  4. 弹出选择框，选择已完成的行
【实际结果】确认框关闭后，鼠标焦点消失了
【期望结果】焦点应自动回到扫码框
【截图】(附图)
```

### 提交渠道

- **即时沟通**: 工作群直接发消息 (附截图)
- **正式记录**: 填写反馈模板发邮件或录入问题跟踪系统
- **紧急问题**: 直接联系系统管理员
